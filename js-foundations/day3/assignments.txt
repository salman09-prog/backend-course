SECTION A — Concept Checks (5 questions)

Q1. What is the exact difference between for…of and for…in?
ANS = for…of is used to iterate over arrays,string,iterables. for…in is used to iterate over object keys, array indexes as strings

Q2. Why does forEach NOT allow break?
ANS = forEach uses a callback function, and the loop itself is not in your control.
You can’t break out because the callback is not the loop — it’s just a function call executed by the loop internally

Q3. What is the main advantage of a do…while loop over a while loop?
ANS = do…while loop executes the task atleast once ,then it checks the condition, whereas while loop first checks the condition then executes the block of code.

Q4. What happens if you write this? 
for (let i = 0; i <= arr.length; i++) { ... }

ANS = It will give us undefined since i <= arr.length, it will try to extract the value of the index which doesnt exist

Q5. Why is this wrong?
for (const key in arr) {
    console.log(arr[key]);
}
ANS = for in loop iterates over index in case of arrays not the elements, and it also accesses the prototypes also, so it would result in unexpected errors.
